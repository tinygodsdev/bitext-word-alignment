# Rules for writing client-facing texts

When writing any **client-facing texts** (content pages, landing and other marketing pages, instructions, etc.) strictly follow the rules bellow.

- Trust readers. State facts directly. Skip softening, justification, hand-holding.
- Avoid binary contrasts, negative listings, dramatic fragmentation. Don't ever use phrases like "This is not X, this is Y", "It's not about X, it's about Y", "It's not just X, it's Y", "not only X, but also Y", etc. 
- Vary rhythm. Mix sentence lengths. End paragraphs differently. Some sentences can be longer, others may be short. Make sure the text doesn't sound monotone. 
- Don't ever use em dash. If you encounter a context, where em dash should be used, use other punctuation or rewrite so that em dash won't be needed.
- Don't use sycophantic language like "great question!", "What a precise observation!", etc. 
- Don't use something like "deep dive", "dive in", etc. 
- Don't use triads unless strictly required by the topic/domain knowledge. Prefer two or one items. 
- Avoid adverbs and other words that don't carry a specific factual meaning. Some examples of what should be avoided: "really", "just", "literally", "genuinely", "honestly",  "simply", "actually", "deeply", "truly", "fundamentally", "inherently" and so on. 
- Don't user filler phrases and rhetorical setups like "what if I told you...", "In today's fast-paced world...", "Here's what I mean:", "Think about it:", "And that's okay.", "At its core", "In today's [X]", "It's worth noting", "At the end of the day", "When it comes to", "In a world where", "The reality is", etc. 
- Avoid unnecessary hedging and bothsidesism (false balance). Factual trade-off are ok. 
- Avoid hype language like "What if I told you", "Have you ever heard", "Imagine this:", "Game-changer", "unlock the potential", etc. 
- Avoid bolded lead-in labels (Word: then a sentence)
- Avoid words that doesn't bring any factual meaning: "comprehensive," "robust," "crucial," "navigate," "utilize," "nuanced", etc. Replace them with evidence or drop completely. 
- Remove self-referential asides and meta-commentary like: "Hint:", "Plot twist:" / "Spoiler:", "You already know this, but", "Dressed up as", "The rest of this essay explains...", "Let me walk you through...", "In this section, we'll...", "As we'll see...", "I want to explore..."


## Don't use business jargon and corporate language. 

Replace with plain language.

| Avoid | Use instead |
|-------|-------------|
| Navigate (challenges) | Handle, address |
| Unpack (analysis) | Explain, examine |
| Lean into | Accept, embrace |
| Landscape (context) | Situation, field |
| Game-changer | Significant, important |
| Double down | Commit, increase |
| Deep dive | Analysis, examination |
| Take a step back | Reconsider |
| Moving forward | Next, from now |
| Circle back | Return to, revisit |
| On the same page | Aligned, agreed |

Note: jargon of other domain fields can be used if it is clearly relevant for the audience. 

## Limitations

This doesn't apply to the code, comments or internal docs.
